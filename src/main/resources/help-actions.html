Actions

Close

<close />
Close the outputstream of the response (not really a disconnect).

Delay

<delay ms="1000" />
Make the proxy wait for a number of milliseconds during processing of the request.

Forward

<forward />
Forward the request to the host as defined by the request.

Host

<host value="localhost" />
Change the host of the request. Forward will deliver the request to that host.

Respond

<respond />
Send the response back to the application that sent the request.

ResponseBody

<responsebody>
    <body>         
      <![CDATA[
      <html></html>
      ]]>
    </body>
</responsebody>
Set the response body that will be send back to the client.

ResponseHeader

<responseheader remove="Date" add="X-TEST" value="Y" />
Remove or add the specified Header with the given value.

Status

<status code="500" />
Set the status code of the response that will be send back to the client.

Record

<record />
Store the complete request and response including headers. Accessible via /recordings

Trace

<trace />
Dump information about the request or response (depends on location w.r.t forward).

Script

<script>
    <body>         
      <![CDATA[
      critter.log("executing script");
      critter.respondWith(200,"world");
      ]]>
    </body>
</responsebody>
Execute the Javascript to process a request/response. The variable "critter" can be used to access information. See Javadoc com.philemonworks.critter.rule.ScriptContext for the available API.
