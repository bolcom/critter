Definitions

ProtocolBuffers message are transported in a compact binary format. The format does not include metadata about the values; field names of the structure is not part of the message. In order for Critter to interpret the message, the definition of that message (including all its dependencies) must be registered. Registration of message definitions is done using a REST api ; it is (currently) not part of the rule definition. 

  • POST /rules/{id}/proto

    Register all message definitions by sending the contents of the .proto file. 

  • GET /rules/{id}/proto/{messageType}

    Return the definition of a messageType registered for a given rule. 

  • GET /rules/{id}/proto

    Report on all registered message types and provide examples to access their values.
